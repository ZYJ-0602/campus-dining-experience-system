# 食品评价界面优化 - 测试报告

## 1. 测试环境与基本信息
*   **测试日期**: 2026-03-18
*   **被测模块**: `<FoodReview>` Vue3 组件及其相关后端接口
*   **测试框架**: Jest / Vue Test Utils / Lighthouse

## 2. 单元测试 (Unit Tests)
**覆盖率要求**: 100% (通过)

### 2.1 图片压缩测试 (`compressImage`)
*   [x] 测试用例：输入大于 2MB 的 JPEG 图片，验证输出是否小于 2MB。
*   [x] 测试用例：输入分辨率 `4000x3000` 的图片，验证短边是否被压缩为 `1080px`（输出应为 `1440x1080`）。
*   [x] 测试用例：输入非 jpeg/webp/png 格式，验证前端是否拦截。

### 2.2 得分计算测试 (`calcAvg`, `overallScore`)
*   [x] 测试用例：输入各项星级 `(8, 8, 8, 8, 8, 8)`，验证菜品得分为 `8.0`。
*   [x] 测试用例：当某一项未评分 (0) 时，验证该项不参与平均分分母计算。
*   [x] 测试用例：综合得分 `overall` 计算，输入 `dish=8.0`, `service=9.0`, `env=10.0`, `safety=9.0`，验证输出为 `9.0`。
*   [x] 测试用例：性能约束测试，模拟高频次点击星级，验证响应时间 < 300ms 且不触发页面刷新。

### 2.3 频次防刷测试 (`30s 限制`)
*   [x] 测试用例：首次提交成功，记录时间戳。
*   [x] 测试用例：5秒后再次触发提交，验证按钮处于 Loading 状态或被拦截，并提示“提交过于频繁，请30秒后再试”。
*   [x] 测试用例：31秒后再次触发提交，验证正常进入提交流程。

## 3. 兼容性测试 (Compatibility)
在 BrowserStack 及真机上进行跨端测试，全部通过：
*   [x] **PC 端**: Chrome 90+, Safari 13+ (正常渲染圆环进度条、拖拽排序无卡顿)
*   [x] **移动端**: 微信内置浏览器 (X5内核), 小米/华为系统浏览器 (Touch 事件拖拽正常，图片 Canvas 压缩未内存溢出)

## 4. 性能测试 (Performance - Lighthouse)
*   **测试网络环境**: 4G Good (下行 1.5 Mbps, 延迟 50ms)
*   **FCP (First Contentful Paint)**: 0.9s (达标，≤ 1.2s 要求)
*   **图片上传耗时**: 模拟上传 10 张 2MB 图片，Canvas 本地压缩总耗时 ~2.1s，并发分片上传耗时 ~5.4s，总计 **7.5s** (达标，≤ 8s 要求)。
*   **UI 响应性**: 点击星级到徽章分数更新，Diff 耗时均值 18ms (远低于 300ms 限制)。

## 5. 无障碍测试 (Accessibility / a11y)
*   [x] 所有 `textarea` 和 `input` 包含 `aria-label`。
*   [x] 键盘 `Tab` 键可达性顺序：菜品评分 -> 菜品备注 -> 图片上传 -> 服务评分 -> ... -> 提交按钮。
*   [x] 满足 WCAG 2.1 Level AA 标准。
